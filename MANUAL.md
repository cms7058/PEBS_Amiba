## 1. 角色与权限

系统支持 3 种角色，决定能看见哪些菜单：

| 角色 | 中文 | 可见菜单 | 适用 |
|---|---|---|---|
| `admin` | 管理员 | 全部 + 系统管理（用户 / 题库 / 模型与设置） | 运维 / 总部 |
| `consultant` | 顾问 | 总览 / 四引擎 / 企业画像 / 移动端 | 主要使用者 |
| `viewer` | 查看者 | 同 consultant，但敏感操作受限 | 老板 / 旁观角色 |

### 1.1 登录方式

**首选：邮箱 + 邀请码**（云函数验证）
1. 打开 `/login`
2. 默认在「邮箱 + 邀请码」Tab
3. 填入登记邮箱 + PEBS 内测邀请码 → 验证并进入
4. 如果邮箱未注册，系统自动提示 → 点"申请邀请码"打开 `https://lingcan.pebs.online/#/pages/copilot/index`

**备用：管理员密码**（本地账号，运维用）
1. 切到「管理员密码」Tab
2. 默认 `admin` / `admin123`（首次部署）
3. 登录后立即去 `/admin/users` 改密码

### 1.2 登出
右上角头像 → "退出登录"。

**操作要点**：

## 3. 诊断引擎

URL：`/diagnosis`
适用：顾问 / 企业管理员

### 3.1 三步流程

**Step 1 — 选企业**
- 列表显示你有权访问的全部企业
- 若是新客户 → 点右上「新建企业」（弹窗填：名称 / 行业 / 规模 / 联系人）
- 已有则直接选

**Step 2 — 选会话**
- 一家企业可以有多份诊断历史（不同时间）
- 已完成会话：可重温对话原文、下载 MD
- 进行中：可继续
- 没有"进行中"时，右上「新开诊断」按钮亮起

**Step 3 — 对话**
- 卡片式问答（单选 / 多选 / 数字 / 短文本 / 长文本 5 种）
- 一次只回答 1-2 题，无需手动打字
- 每答完一题，**自动保存到云端**（关浏览器也不丢）
- 右侧六维进度条实时刷新；完成 100% 显示绿色 ✓

### 3.2 完成诊断

- AI 觉得问得差不多了 → 主动给出 `done` 卡片 + 综合画像 → **系统自动 finalize**，画像同步到企业档案
- 若想中途提前结束 → 右上「标记完成」按钮（手动 finalize）
- finalize 后右上按钮变为「重新生成画像」，随时可重跑（会更新长期记忆）

### 3.3 导出 / 重温
- 右上「导出 MD」生成完整 markdown 报告（含企业概况 / 六维覆盖度 / 优势 / 风险 / 决策 / 全部对话原文 / 长期记忆）
- 适合给客户出具诊断报告 / 内部存档

## 4. 企业画像

URL：`/profile`
适用：所有角色

### 4.1 顶部
- 综合就绪度（0-100）/ 推荐档次（L1/L2/L3）/ 预计周期
- 行业 + 规模 + 联系人

### 4.2 阿米巴落地全景思维导图（核心）
- 中心节点 = 企业
- 7 大分支：现状 / 去除 / 改造 / 新建 / 规则 / 步骤 / 风险
- **点击任意节点** → 下方展开详情卡（数据源 / 责任人 / 时间 / 预算 / 风险 / 缓解措施） + 嵌入图表（柱图 / 饼图 / 公式 / 进度条 …）
- 同时右下 AI 助手按钮自动同步该节点为讨论上下文

### 4.3 六维诊断神经图
- 中心 6 节点 = 六维（按得分自动配色：绿 ≥75 / 蓝 ≥60 / 琥珀 ≥45 / 红 <45）
- 外圈 18 节点 = L1/L2/L3 子问题
- Hover 看连接 / 点击看详情

### 4.4 问题关联图
- 4 列流：优势 → 风险 → 影响维度 → 决策点
- AI 自动按关键词连线，让你直观看出"哪个风险拖了哪个维度，对应哪个待决策"

### 4.5 同行业基准对比
- 综合就绪度百分位（"超过 X% 同行"）
- 六维分项 vs 行业均值双柱图
- L1/L2/L3 落地档次分布

### 4.6 优势 / 风险 / 决策点
卡片化呈现，颜色区分。

适用：顾问 / 企业管理员

## 8. AI 助手（页内浮动）

适用：企业画像 / 规划 / 设计 / 部署 4 个页面右下角浮动按钮

### 8.1 三个快捷动作
| 按钮 | 用途 |
|---|---|
| **解释当前节点** | 3 段话解释概念 / 必要性 / 落地建议 |
| **生成 CSV 模板** | AI 输出可直接下载的工作底稿 .csv |
| **找相关资料** | 5 条学习方向 + 关键词 + 可信度 |

### 8.2 上下文感知
- 点页面任意图谱/表格的节点 → 助手按钮立刻显示该主题徽章
- 打开抽屉时 header 显示「AI 记忆中：X」+「待切换：Y」两个徽章
- 上下文变化时如果对话已有历史，会弹**切换确认横幅**：
  - 「继续讨论 X」— 保留对话历史
  - 「切换到 Y 并清空对话」— 重新开始，避免新主题被旧上下文污染

### 8.3 提问技巧
- 直接说"帮我生成一个 XX 工作表" → AI 输出 CSV → 一键下载
- 说"找一些关于 阿米巴 内部转让定价 的资料" → AI 给学习方向
- 说"为什么这个改造要 80 万？" → AI 拆解预算逻辑

## 10. 系统管理（仅 admin）

URL：`/admin/users`、`/admin/questions`、`/settings`

### 10.1 用户管理
- 列出全部账号（含邀请用户和管理员）
- 新建用户：用户名 / 显示名 / 角色 / 密码（≥6 位）
- 编辑：改显示名 / 角色 / 密码（留空 = 不改）
- 删除：不能删自己 / 系统保证至少留 1 个 admin

### 10.2 题库与建议
两个 Tab：

**题库 Tab**
- 按 6 维分组列出全部题目
- 区分来源：内置（灰）/ 管理员（蓝）/ AI 建议（琥珀）
- 编辑 / 删除 / 新增（支持 5 种 type：single / multi / number / text / textarea）
- 状态可改：active / draft / archived

**待审建议 Tab**
- 智能体在对话中识别的"题库未覆盖但重要"的现象会自动进这里
- 每条显示：维度 / 层级 / 类型 / 题面 / AI 给出的理由 / 来源会话
- 审核：编辑文案 → 批准（自动入题库为 active） / 拒绝
- ⚠️ **AI 永远不会自动入库**，必须 admin 审批

### 10.3 模型与设置
- 配置 3 个大模型的 API 接入：
  - DeepSeek V4 Pro（OpenAI 协议）
  - MiniMax M2.7（Anthropic Messages 协议）
  - Kimi K2.5（OpenAI 协议）
- 每个模型可改：协议 / Base URL / Model 名 / API Key
- 设默认模型（诊断对话 / AI 助手都用这个）
- 测试连接：点"测试连接"按钮，发一句"你好"看是否回应

API Key 仅存浏览器 localStorage，不上传服务器，不入库。

## 11. 常见问题（FAQ）

### Q1 诊断完成了但企业画像页显示"尚无完成的诊断"
- 原因：旧版本的诊断未触发 finalize
- 解决：打开诊断引擎 → 选该企业的已完成会话 → 右上点「重新生成画像」一次
- 已修复：新版会在 AI 给出 done 卡片时自动 finalize

### Q2 AI 助手回复了一通莫名其妙的内容
- 多半是上下文被旧节点污染。检查抽屉 header：
  - 如果有"待切换"徽章 → 点上方横幅"切换并清空对话"
  - 如果不想清空 → 选"继续讨论"，让 AI 知道仍在旧主题

### Q3 思维导图节点文字被截断
- 已修复（节点宽度按中文字符数自适应）
- 如果还有：刷新页面（Cmd+Shift+R 强刷）

### Q4 MiniMax 调用提示 502
- 原因：协议选错了
- 解决：模型与设置 → MiniMax → 协议选 **Anthropic Messages**，不是 OpenAI 兼容

### Q5 流式响应不打字机（要等完）
- Nginx/反代缓冲了 SSE
- 解决：反代配置加 `proxy_buffering off;` 和 `proxy_request_buffering off;`
- 详见 [DEPLOY.md 三、配置反向代理](DEPLOY.md#三配置反向代理--https生产必备)

### Q6 怎么让客户也能进来用
- admin 在「用户管理」给客户开账号（角色选 consultant / viewer）
- 或让客户去 `https://lingcan.pebs.online/#/pages/copilot/index` 申请邀请码

### Q7 数据备份
- 命令：`./deploy.sh backup`（生成 amoeba-backup-YYYYMMDD.tgz）
- 建议设 crontab 每日凌晨备份，保留 7 天
- 详见 [DEPLOY.md 五、数据持久化](DEPLOY.md#五数据持久化)

### Q8 想看完整对话历史
- PC：诊断引擎 → 选企业 → 选会话 → 看对话区 + 右上「导出 MD」
- 手机：`/m/conversations/[id]`

### Q9 升级版本会丢数据吗
- 不会。所有数据在 Docker 命名卷 `amoeba-data` 中（`/app/data/`），与镜像分离
- `git pull && ./deploy.sh` 升级时数据卷自动保留

### Q10 默认 admin 密码忘了
- 应急：进容器 `docker exec -it amoeba-copilot sh`，删 `/app/data/users.json`，重启容器（会用 `AMIBA_ADMIN_PASSWORD` 重新 seed）
- ⚠️ 这会丢失全部用户记录，慎用

## 12. 典型使用流程

### 流程 A：第一次给一家客户做诊断
1. PC 登录 → 诊断引擎 → 「新建企业」填基本信息
2. 「新开诊断」→ 卡片式回答 20-30 分钟 → AI 自动 finalize
3. 切到企业画像 → 看综合就绪度 + 全景思维导图 + 同行业对比
4. 在画像页点任一思维导图节点 → 让 AI 助手生成 CSV 工作底稿 → 下载
5. 出诊断报告：诊断引擎页右上「导出 MD」

### 流程 B：把一个改造任务落地
1. 企业画像 → 思维导图 → 点击「改造」分支下的某节点（如"辅料出库改扫码"）
2. AI 助手按钮自动显示该主题 → 打开抽屉 → 点「生成 CSV 模板」
3. AI 输出含字段定义和示例行的 CSV → 下载 → 发给仓储/IT 负责人填写
4. 切到部署引擎 → 实施任务清单查看进度

### 流程 C：日常运行监控
1. PC 首页 → 看预警卡 → 点击任一预警进入详情（含时间线 / 影响范围 / 行动建议）
2. 切到部署引擎 → 选企业 → 看各阿米巴运行台账 → 展开问题阿米巴看完整明细
3. 手机扫码 → `/m` → 随时查看最新评分和会话

### 流程 D：管理员审批 AI 建议的新题目
1. admin 登录 → 系统管理 → 题库与建议 → 待审建议 Tab
2. 逐条查看：题目 / AI 理由 / 来源会话
3. 编辑文案（可改）→ 批准 → 自动入题库 → 后续诊断会用到
