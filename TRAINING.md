# 阿米巴 × AI：从经营哲学到企业落地

> 一日特训课程大纲 · 上海零参科技 · Amoeba Copilot
> 时长：1 天（上午 3 小时 + 下午 3.5 小时）
> 对象：制造业企业主、总经理、财务负责人、推行阿米巴的项目负责人
> 形式：方法论讲授 50% + 系统实操 35% + 研讨与行动规划 15%

---

## 课程定位

**这不是软件操作培训。** 本课程回答三个问题：

1. 阿米巴经营为什么在中国制造业落地难？难在哪一层？
2. AI 智能体如何改变阿米巴咨询的成本结构和成功率？
3. 你的企业应该走哪条路径、从哪里开始、避开哪些坑？

系统实操环节使用 Amoeba Copilot 作为教学载体——学员在真实工具中体验"诊断 → 画像 → 规划 → 落地"全过程，把抽象方法论变成可带走的行动方案。

# 上午场（9:00 – 12:00）· 为什么难落地，AI 怎么破局

## 模块 1 · 阿米巴经营的本质与中国式困境（9:00 – 10:00）

**形式：讲授 + 现场自检研讨**

### 1.1 回到原点：阿米巴到底解决什么问题（20 分钟）
- 划小核算单元、内部市场化、全员经营的底层逻辑
- "人人都是经营者"背后的会计学基础：单位时间附加值
- 不是绩效考核工具，而是经营哲学 + 核算体系 + 组织机制的三位一体

### 1.2 中国制造业的落地现实（25 分钟）
- 数据：4000 万+ 中小企业，真正成功导入阿米巴的不足 1%
- **三大现实挑战逐层拆解：**
  - **制度层**：内部定价怎么定才公平？分配规则怎么设不打架？核算粒度到哪一级？
  - **信息化层**：工时数据从哪来？没有 MES 怎么办？ERP 不支持多维核算怎么改？
  - **设备物理层**：共用设备折旧怎么分？能源只有总表怎么计量？辅料账物不符怎么治？
- 典型失败案例画像：轰轰烈烈启动 → 核算算不清 → 分配吵不停 → 半年后名存实亡

### 1.3 现场自检（15 分钟）
- 学员对照三层挑战清单自评：我的企业卡在哪一层？
- 快速举手统计 + 2-3 位学员分享，讲师点评
- **产出：每人一张《落地障碍自检表》**

## 模块 2 · 传统咨询 vs AI 动态智能体（10:10 – 11:10）

**形式：讲授 + 第一次系统实操**

### 2.1 传统咨询模式的结构性缺陷（20 分钟）
- 贵：50–300 万/次，中小企业付不起
- 慢：1–2 年周期，老板等不起
- 模板化：企业套模板，而不是模板适配企业
- 不可持续：顾问撤场即退化，知识没有沉淀在企业里
- 核心论点：**问题不在顾问水平，在服务模式——一次性交付 vs 持续陪跑**

### 2.2 AI 智能体改变了什么（15 分钟）
- 四大能力重构咨询价值链：
  - **诊断能力**：多轮对话式深度探测，比企业自己更快发现问题
  - **设计能力**：基于行业知识库生成适配方案，而非套模板
  - **陪跑能力**：上线后持续监测、预警、给行动建议
  - **进化能力**：每服务一家企业，行业基准和解决方案库就更准一分
- 成本结构对比：5–30 万 + 3–15 个月 vs 传统 50–300 万 + 1–2 年
- 人机分工边界：AI 做数据和规则，顾问做信任和变革——AI 不取代顾问，取代的是低效的服务方式

### 2.3 实操 ①：亲历六维诊断（25 分钟）
> 学员用自己企业的真实情况回答（或用发放的企业背景卡）

- 登录系统 → 新建企业 → 开始卡片式诊断问答
- 体验要点：
  - 六维模型（组织/财务/信息化/设备/流程/文化）的 L1→L2→L3 递进式问询
  - AI 如何根据上一题答案动态调整下一题
  - 文化维度的"行为式提问"：不问"文化好不好"，问"上次员工主动提改善建议是什么时候"
- **思考题：这些问题如果由顾问来问，要几天？要多少钱？**

✅ 检查点：每位学员完成 15 题以上，六维进度条至少 3 个维度过半。

## 模块 3 · 读懂企业画像：把模糊感觉变成可决策数据（11:10 – 12:00）

**形式：实操 + 小组研讨**

### 3.1 实操 ②：解读自己的企业画像（25 分钟）
- 综合就绪度评分（0-100）意味着什么：60 分该不该启动？
- 推荐档次 L1 / L2 / L3 与投入产出的对应关系
- **阿米巴落地全景思维导图**——一张图看清全貌：
  - 七大分支：现状盘点 / 要去除 / 要改造 / 要新建 / 核算规则 / 实施步骤 / 风险管控
  - 颜色语言：灰=沿用、红=去除、琥珀=改造、蓝=新建、紫=规则

# 下午场（13:30 – 17:00）· 硬骨头怎么啃 + 我的落地路线图

## 模块 4 · 核算规则：阿米巴公平性的基石（13:30 – 14:40）

**形式：讲授 + 实操**

### 4.1 五大核算难题与解法（40 分钟）

| 难题 | 解法 | 关键公式 / 机制 |
|---|---|---|
| 内部价格怎么定 | 内部转让定价 | 转让价 = 标准工时成本 × (1 + 内部利润率%)，季度核定、期间冻结 |
| 共用设备怎么分 | 设备内部租赁制 | 设备工时单价 = (月折旧 + 月均维修 + 备件储备金) ÷ 月度可用工时 |
| 能源费用怎么算 | 三级计量体系 | 高耗能精确计量 / 中耗能工时分摊 / 低耗能面积分摊 |
| 辅料失控怎么治 | 定额消耗法 | 超标 10% 以内阿米巴自担，超 10% 触发异常分析 |
| 质量损失谁承担 | 责任归因矩阵 | 操作失误 100% 责任方 / 工艺超负荷工程制造各 50% / 客诉按责任比例分摊 |

- **核心认知：不追求 100% 精确**——直接成本 95%+ / 间接成本 80% / 隐性成本 60-70% / 管理成本只要趋势正确
- 浪费成本显性化：废品、返工、超耗、空转、加班、客诉、处置七类浪费如何进入阿米巴利润表

### 4.2 实操 ③：生成自己企业的核算工作底稿（30 分钟）
- 在规划引擎「核算规则」页精读公式 + 应用示例
- 点击「内部转让定价公式」→ 打开 AI 助手 → 生成 CSV 工作底稿并下载
- 追问 AI："我们有 X 个车间 / 项目组，帮我扩展这张表" —— 体验上下文感知
- **产出：每人 1 份可直接回企业试填的《内部转让定价底稿.csv》**

✅ 检查点：每人下载至少一份 AI 定制的工作底稿。

## 模块 5 · 数据从哪来：信息化的务实路线（14:40 – 15:40）

**形式：讲授 + 实操**

### 5.1 三档投入，量力而行（25 分钟）

| 档次 | 工具 | 数据频率 | 改造成本 | 适合 |
|---|---|---|---|---|
| L1 基础版 | Excel + 钉钉/企微 | 周报手工 | 近零成本，3 个月可跑 | 先跑起来再说 |
| L2 标准版 | ERP 改造 + 简易 IoT | 日报自动 + 月度补录 | 20–80 万，6 个月 | 大多数中型企业 |
| L3 高级版 | MES+ERP+IoT 全打通 | 实时采集，T+1 核算 | 100–300 万，12 个月 | 信息化基础好的企业 |

- 三条实施路径怎么选：A 快速见效型（6-9 月）/ B 稳健推进型（12-18 月）/ C 数字原生型
- 选型决策树：预算上限 × 老板耐心 × 组织变革承受力 × 信息化起点
- 常见误区：先上 MES 再推阿米巴？——正确答案多数是"手工台账 + MES 并行"

### 5.2 实操 ④：从财务明细反推数据需求（35 分钟）
- 部署引擎 → 展开一个制造阿米巴的完整运行台账
- 逆向思考练习：面板上的每个数字，数据从哪来？
  - 员工工时表 ← MES 工位刷卡 / 手工台账
  - 设备 OEE ← IoT 状态机 / 班组记录
  - 能耗明细 ← 智能电表分级 / 总表估算
  - 辅料定额对比 ← 扫码出库 / Excel 周报
  - 浪费成本 7 类 ← 质量系统 + 工时 + 超耗联合计算
- 点开「数据/信息化系统架构图」：数据源层 → 中台引擎层 → 应用看板层，点击节点看对接方式
- **产出：每人一张《我的企业数据缺口清单》（哪些数据现在就有 / 哪些要改造 / 哪些要新建）**

## 模块 6 · 组织与人：制度设计与风险管控（15:50 – 16:30）

**形式：讲授 + 案例研讨**

### 6.1 比算账更难的是人（25 分钟）
- 阿米巴长任命与考核：技术 + 财务双线标准，任期制
- 决策权下放的刻度：10 万以内阿米巴长定 / 10–50 万联签 / 50 万以上董事会
- 超利分享机制：超出目标 110% 的部分，30% 进分红池、70% 留企业——为什么是这个比例
- 退出与降级机制：连续 2 季度不达标降级、3 季度重组——没有退出机制的阿米巴是假阿米巴
- 跨阿米巴争议处置：先协商 → 总部仲裁限 1 周 → 沉淀进规则库

### 6.2 典型风险与缓解（15 分钟）
- 风险矩阵读法：概率 × 影响，右上角优先治理
- 三个高频风险的实战对策：
  - 中层抵触 → 提前一对一沟通 + 试点示范，不搞一刀切
  - 数据双轨期混乱 → 并行至少 3 个月、对账误差 < 5% 才切换
  - 定额初版不准 → 首季宽松定额，季末按实际修正——完美主义是落地杀手

## 模块 7 · 结业：我的企业落地路线图（16:30 – 17:00）

**形式：个人产出 + 答疑**

### 7.1 一页行动计划（20 分钟）
每位学员基于全天产出（自检表 / 画像 / 工作底稿 / 数据缺口清单），完成一页《90 天落地路线图》：

3. **90 天三个里程碑**：第 30 天 / 60 天 / 90 天各完成什么
4. **需要的外部支持**：顾问介入深度、信息化改造预算区间

### 7.2 集中答疑 + 资源发放（20 分钟）
- 高频问题集中解答
- 资源包：课程讲义 / 操作手册 / 设计方案全文 / 今日个人全部产出（诊断报告 MD + 工作底稿 CSV）
